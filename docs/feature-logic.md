\# Tarot Web App 功能逻辑文档



\## 1. 文档目标



\* 定义塔罗牌抽牌功能的程序逻辑

\* 定义正位与逆位生成逻辑

\* 定义页面状态更新逻辑

\* 定义解读内容生成逻辑

\* 为前端实现提供明确规则



\## 2. 功能范围



\* 用户输入问题

\* 系统校验输入内容

\* 系统随机抽取三张不重复的牌

\* 系统为每张牌生成正位或逆位

\* 系统生成抽牌结果

\* 系统更新页面状态

\* 系统展示牌阵和解读内容

\* 用户可重置页面



\## 3. 输入逻辑



\### 3.1 用户输入问题



\* 用户在输入框中输入问题

\* 输入内容保存到 question 状态



\### 3.2 输入校验



\* 用户点击开始占卜时触发校验

\* 系统对 question 执行 trim 处理

\* 如果 question 为空字符串

\* 设置 error 为 true

\* 停止后续抽牌流程

\* 如果 question 不为空字符串

\* 设置 error 为 false

\* 进入抽牌流程



\## 4. 抽牌逻辑



\### 4.1 牌库来源



\* 牌库来源于本地 tarotCards 数据数组

\* 牌库总长度为 78

\* 每张牌在数组中唯一存在一次



\### 4.2 抽牌规则



\* 每次占卜固定抽取 3 张牌

\* 3 张牌不可重复

\* 抽牌顺序对应固定位置

\* 第 1 张为 past

\* 第 2 张为 present

\* 第 3 张为 future



\### 4.3 抽牌实现思路



\* 复制一份 tarotCards 数组

\* 对复制后的数组执行随机打乱

\* 取打乱后数组的前 3 个元素

\* 将这 3 张牌依次分配给 past present future



\## 5. 随机逻辑



\### 5.1 洗牌逻辑



\* 使用随机洗牌算法打乱卡牌顺序

\* 洗牌后每张牌都有机会出现在任意位置

\* 洗牌结果用于抽取前 3 张牌



\### 5.2 去重逻辑



\* 由于先整体洗牌再截取前 3 张

\* 因此天然保证抽到的 3 张牌不重复



\## 6. 正位与逆位逻辑



\### 6.1 生成规则



\* 每张牌在抽取后单独生成一次随机状态

\* 状态值为 reversed

\* reversed 为 true 表示逆位

\* reversed 为 false 表示正位



\### 6.2 显示规则



\* reversed 为 false 时 图片正常显示

\* reversed 为 true 时 图片旋转 180 度显示



\### 6.3 数据结果



\* 每张抽中的牌都需要附加 reversed 字段

\* 每张抽中的牌都需要附加 position 字段



\## 7. 抽牌结果结构



\### 7.1 单张结果结构



\* position

\* card

\* reversed



\### 7.2 完整结果结构



\* question

\* cards



\### 7.3 三张牌位置定义



\* past 对应过去

\* present 对应现在

\* future 对应未来



\## 8. 解读生成逻辑



\### 8.1 第一版实现方式



\* 第一版不依赖外部 API

\* 使用本地模板文本生成解读内容



\### 8.2 单牌解读规则



\* 如果 reversed 为 false 使用 meaningUpright

\* 如果 reversed 为 true 使用 meaningReversed



\### 8.3 综合解读规则



\* 将 question 与三张牌的结果组合

\* 按过去 现在 未来顺序拼接解读文本

\* 输出为 interpretation 字符串



\### 8.4 后续升级方式



\* 后续可将 question 和 cards 发送给 AI 接口

\* 由 AI 生成更自然的完整解读文本



\## 9. 页面状态更新逻辑



\### 9.1 初始状态



\* question 为空

\* error 为 false

\* isDrawn 为 false

\* cards 为空数组

\* interpretation 为空字符串



\### 9.2 抽牌成功后



\* error 设置为 false

\* cards 更新为新的三张牌结果

\* interpretation 更新为新的解读文本

\* isDrawn 设置为 true



\### 9.3 输入错误时



\* error 设置为 true

\* cards 不更新

\* interpretation 不更新

\* isDrawn 保持 false



\## 10. 重置逻辑



\### 10.1 触发方式



\* 用户点击重新开始按钮



\### 10.2 重置内容



\* 清空 question

\* 清空 error

\* 清空 cards

\* 清空 interpretation

\* 设置 isDrawn 为 false



\### 10.3 重置结果



\* 页面回到初始状态

\* 隐藏牌阵区域

\* 隐藏解读区域

\* 隐藏重置按钮



\## 11. 功能执行流程



\### 11.1 开始占卜流程



\* 用户输入问题

\* 用户点击开始占卜

\* 系统校验输入

\* 输入为空则显示错误提示并结束流程

\* 输入有效则开始抽牌

\* 系统洗牌

\* 系统取前 3 张牌

\* 系统为每张牌生成正逆位

\* 系统生成解读文本

\* 系统更新页面状态

\* 页面显示牌阵和解读



\### 11.2 重新开始流程



\* 用户点击重新开始

\* 系统清空所有状态

\* 页面恢复初始显示



\## 12. 第一版实现限制



\* 固定使用三张牌阵

\* 不支持多牌阵切换

\* 不支持用户历史记录

\* 不支持联网抽牌

\* 不支持 AI 对话式追问



\## 13. 后续可扩展方向



\* 支持单张牌阵

\* 支持更多牌阵模式

\* 支持卡牌翻转动画

\* 支持 AI 生成高级解读

\* 支持结果保存

\* 支持分享功能
