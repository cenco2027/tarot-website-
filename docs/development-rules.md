\# Tarot Web App 开发规则文档



\## 1. 文档目标



\* 统一项目开发规范

\* 提高代码可读性与可维护性

\* 规范文件结构与命名方式

\* 为 Cursor 提供稳定开发上下文



\## 2. 技术栈约定



\* 前端框架 React

\* 语言 TypeScript

\* 构建工具 Vite

\* 样式方式 CSS 或简单模块化 CSS

\* 不引入复杂状态管理库



\## 3. 项目目录结构



\* public 用于存放静态资源

\* src 用于存放源代码



目录结构



\* public



&#x20; \* images



&#x20;   \* backgrounds

&#x20;   \* cards



\* src



&#x20; \* components

&#x20; \* pages

&#x20; \* data

&#x20; \* styles

&#x20; \* utils



\## 4. 文件命名规则



\### 4.1 组件命名



\* 使用 PascalCase

\* 文件名与组件名一致



示例



\* TarotCard.tsx

\* DrawButton.tsx



\### 4.2 普通文件命名



\* 使用小写英文

\* 使用短横线连接



示例



\* tarot-cards.ts

\* draw-logic.ts



\### 4.3 图片命名



\* 使用英文命名

\* 使用短横线

\* 保持统一格式



\## 5. 组件开发规则



\### 5.1 单一职责



\* 每个组件只负责一个功能

\* 不在组件中混合多种逻辑



\### 5.2 组件拆分



\* 页面由多个小组件组成

\* 避免一个文件过大



\### 5.3 Props 规则



\* 明确声明类型

\* 不传递多余数据

\* 保持结构清晰



\## 6. 状态管理规则



\### 6.1 状态位置



\* 状态集中在页面组件

\* 子组件通过 props 获取数据



\### 6.2 状态字段



\* question

\* error

\* isDrawn

\* cards

\* interpretation



\### 6.3 状态更新



\* 使用函数更新状态

\* 不直接修改原数据



\## 7. 数据使用规则



\### 7.1 数据来源



\* 所有卡牌数据来自 src/data/tarotCards.ts



\### 7.2 不允许行为



\* 不在组件中硬编码卡牌数据

\* 不重复定义数据结构



\## 8. 抽牌逻辑规则



\### 8.1 逻辑位置



\* 抽牌逻辑放在 utils 或独立逻辑文件中



\### 8.2 不允许行为



\* 不在 UI 组件中写随机逻辑

\* 不在多个地方重复实现抽牌



\## 9. 样式规则



\### 9.1 样式位置



\* 样式写在 styles 文件夹或组件内简单样式



\### 9.2 样式原则



\* 保持简洁

\* 避免内联样式过多

\* 使用统一颜色体系



\### 9.3 不允许行为



\* 不使用高复杂 UI 框架

\* 不使用不一致的颜色



\## 10. 代码风格



\### 10.1 基本规范



\* 保持缩进一致

\* 保持结构清晰

\* 避免过长函数



\### 10.2 函数规则



\* 函数命名清晰

\* 函数职责单一



\## 11. 逻辑与 UI 分离



\### 11.1 原则



\* 逻辑代码与 UI 渲染分离



\### 11.2 实现方式



\* 逻辑写在 utils 或 hooks 中

\* UI 组件只负责展示



\## 12. 错误处理规则



\### 12.1 输入错误



\* 使用 error 状态控制显示



\### 12.2 不允许行为



\* 不使用 alert

\* 不阻断页面



\## 13. 开发流程



\* 编写文档

\* 根据文档拆分组件

\* 实现数据结构

\* 实现逻辑

\* 实现 UI

\* 联调与测试



\## 14. 第一版开发范围



\* 单页面应用

\* 三张牌抽牌

\* 本地数据

\* 本地解读

\* 无用户系统



\## 15. 后续扩展规则



\* 保持现有结构不破坏

\* 新功能独立模块开发

\* 不修改核心数据结构

\* 不影响现有逻辑流程


\## 16. 命名规范单一来源

\* 卡牌 key 与图片命名统一以 docs/naming-source-of-truth.md 为准

\* key 必须与图片文件名去后缀一致

\* image 必须使用 /images/cards/${key}.jpg


\## 17. 校验脚本与自测要求

\* 每次改动卡牌数据后运行 npm run check:cards

\* 自测清单统一参考 docs/self-test-checklist.md
